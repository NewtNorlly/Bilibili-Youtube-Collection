
各位暴风中的旅行者你们好，我是木子。欢迎来到生命管理课程的第三期。我们今天继续来讨论一下知识管理的相关理论。在上个视频里头呢我们聊到了 code 流程的前两个部分，<span style="background:#fff88f">capture 捕捉和 organize 整理</span>。

那么接下来的第三部分是 distill 提炼，提炼部分的关键词是重构。我们记笔记的本质是透过时间向未来的自己发送各种知识包。如果你给未来的自己发了一个 100 页的 pdf，那么大概率是没有人会看的。就好像很多人读到一篇好的文章很喜欢啊，第一反应都是全文收藏。但<span style="background:#fff88f">简单的复制粘贴是一种没有营养的搬运</span>。我们更需要的是通过<span style="background:#fff88f">重构信息</span>，用<span style="background:#fff88f">自己的语言和理解来改写</span>每一个学到的知识点。举个例子啊，你最近在学习矢翠山效应这个知识点，网上给这个概念的定义是试图阻止大众了解某些信息或压制特定的网络信息，结果适得其反，反而使该事件为更多的人所了解。然后你看着这个信息，你想了一下，哎，这不就是欲盖弥彰吗？在刚刚这个过程中，你通过现有的思维和词藻来诠释了一个来自外界的信息。所以你平常怎么说话的，知识管理的提炼就应该怎么写，不同的人也有不同的重构方法。

在这里还是为大家推荐三个思路。第一种思路叫<span style="background:#fff88f">原子笔记 custom method</span>，也叫“卡片盒笔记法”。它来自于德国的社会学家和系统理论学家尼克拉斯・卢曼。卢曼一生中写了 58 本书和其他大量的出版物。他个人的独家秘籍就是把单个笔记的内容最小化，聚焦在一个个简单的概念上。遇到大内容，我们就先把大内容打碎成更小的内容，再通过原子笔记之间的双向链接形成一个复杂的思想网络。假设我手上有一篇关于消费主义的长文，我就可以把里面的内容打碎成决策心理学、社交压力、营销案例等各种各样的知识卡片。这里要注意一下，每个卡片里的内容要尽可能的简短，用自己的话来写。第二种思路是赫赫有名的康奈尔笔记法 note taking system。康奈尔笔记会把一个笔记的页面分成三个部分，<span style="background:#fff88f">线索、笔记、总结</span>。在输入阶段，比如说你在上课的时候，我们先把笔记写在笔记区域，这里的内容要尽可能的快速精简。课后可以补充一下刚刚没有写的部分。而到了提炼阶段，我们把笔记的要点用一个个关键词和问题的形式写在线索区域。

接下来就是第三步复习阶段，我们盖住右侧，看着左侧，试图用自己的语言来回答线索里的问题，或者描述线索里提到的各种关键词。最后的最后我们到笔记的总结区域写下最言简意赅的解释。康奈尔系统注重的是一个知识点的持续反刍，非常适合学生党，用来强化自己对考试相关的记忆。最后一种提炼的思路是累进式摘要。<span style="background:#fff88f">Progressive Summarization</span> 还是我们的提阿哥老哥提出的啊。<span style="background:#fff88f">累进式摘要的特点</span>是把一段文字不断的抽丝剥茧，先摘录一句话，然后把其中的重点加粗，重点中的重点进行加亮，然后重点中的重点中的重点用自己的话来总结一遍。这个系统的目的其实和康奈尔很像，也是通过<span style="background:#fff88f">多次审视和浓缩</span>信息来保留<span style="background:#fff88f">最关键</span>的部分。

好，我们终于到达了 code 四部曲的最后一个阶段，那也就是 <span style="background:#fff88f">express 表达</span>，这个部分的关键词是反馈。一个人在表达的时候最好能获得来自外界的某种评价。你可能会觉得反馈有什么用呢？其实它有三个很大的好处，我们来快速过一下。<span style="background:#fff88f">第一，它能够巩固你已经学到的知识</span>。以教为学也被称为费曼学习法。这个很多人都听过，这个概念来自于美国物理学家理查德・费曼。有的时候你在脑子里把一件事情想得已经很清楚了，但你人一开口，整个人突然就卡壳了。通过教授他人，我们被迫以更深层次的方式来处理信息，从而增强了后续的记忆。<span style="background:#fff88f">第二个好处，正向的反馈</span>，比如说别人的夸奖啊，比如说观众正向的评论，<span style="background:#fff88f">都能够增强你继续学习的动力</span>。你大脑里的奖赏系统会被激活，产生愉悦感并分泌多巴胺，鼓励你后续重复类似的行为。你像我个人做了这么久的视频，我做视频所获得的正反馈就是支撑我终身学习的最大支柱。如果我没有做视频这个爱好的话，我绝对不会读这么多书的。<span style="background:#fff88f">第三，表达可以帮助他人</span>，让更多的朋友跟你一起成长。这听起来可能有一点点肉麻，但我们在生活中往往忽略了利他的重要性，总是专注于 “我怎么能够让自己变得更好，我怎么能够让自己变得更厉害”。但帮助他人所带来的幸福感往往更加持久，而且没那么浮躁。周岭在《认知驱动》这本书里提到，<span style="background:#fff88f">人们往往只单方面看到自己想要的，却忽略了自己能给的</span>。帮助他人也往往能帮助你自己。当你的思想能为他人产生价值的时候，别人也会更愿意为你的知识进行付费。

好，讲到这里我们已经覆盖到了提阿格提出的知识管理四部曲：<span style="background:#fff88f">捕捉、整理、提取、输出</span>，和每个阶段对应的关键词：<span style="background:#fff88f">共鸣、关联、重构和反馈</span>。听起来可能还是有点抽象啊，接下来让我们通过一个完整的例子来展示一下你每天可以怎么使用 code。假设你每天都有读书、听播客和记录自己想法的习惯，你有的吧？在捕捉阶段你可以把那些让自己感到共鸣的内容整段摘抄到笔记软件里。当然你也会读到很多不感兴趣的内容，或者你已经开始读一本书了，结果读到一半发现好像没什么意思。如果这些知识没有办法引起你的好奇心，那你就果断扔掉，不需要死缠烂打。这些是来自于外部的信息。那么内部的话，脑子里<span style="background:#fff88f">如果有灵感</span>的时候，你可以拿比如说录音软件把想法记录下来，后续再用<span style="background:#fff88f">听写软件转成文字</span>。好，你的灵感已经全部捕捉完毕了。接下来当你有空的时候，比如说每个周末可以对这些捕捉进来的内容进行大致的分类，通过 PARA 体系将它们挪到对应的文件夹里头，或者是通过双向链接把这些内容绑定到对应的 MOC 上，这样会给你的知识创建一些大致的框架和结构。第三部分是提炼这些已经整理好的知识点，比如说把梳理摘录的内容用累进式摘要进行浓缩和总结，提炼成一两句精华。一边写的时候，你也可以一边想想未来的我会怎么用到这个信息呢？最后你可以把这些吸收好的知识转变成自己的文字，把这些文字分享给家人、朋友，或者是发布到公众号、小红书这样的平台。文章发出去之后，再根据评论区的反馈不断地改进自己的知识循环。你可能也会发现，我需要读更多这类的书，或者我需要听更多这类的播客，然后慢慢的持续改变自己。

Code 系统有一个重要的特质，那就是让你重复接触知识。很多的信息你会在 capture 阶段看一回，organize 阶段看一回，distill 阶段还得再看一回。写作的时候你肯定还是要看一回，这听起来非常的繁琐，但没有办法，重复接触正是建立长期记忆最好的方法。根据德国心理学家赫尔曼・艾宾浩斯的研究，我们在学习后的<span style="background:#fff88f"> 24 个小时内会遗忘大约 74% 的信息</span>。这个研究也非常有名啊，被称为艾宾浩斯遗忘曲线。那么你每天读那么多的信息，如果不及时复习和巩固的话，我们辛苦学习的东西过不了多久就会流失掉。而 code 系统就是一种间隔重复，能加强一个人的记忆。间隔重复的特点就是<span style="background:#fff88f">逐步增加两次复习间的时间间隔</span>，比如当天复习一次，两天后再复习一次，再之后就是一周复习一次，一个月复习一次等等。想要<span style="background:#fff88f">最大程度开发自己的大脑</span>，你就得定期提醒自己，<span style="background:#fff88f">回顾一下以前的内容</span>。间隔重复同时也说明了一个很重要的道理，那就是当你进行知识管理，建立了所谓电子的第二大脑的时候，如果这个大脑最后建的不如你想象中的那么好，其实也没有关系。因为通过不断的接触知识，你在建立第二大脑的同时，其实已经帮助了你的第一大脑。哪怕你的电子笔记不够完美，你自己本人的记忆力也会得到实质性的提升。

除了定期回顾你自己的笔记，我们还有一件可以做的事情，那就是多多参考其他网上大神的笔记。我在这里呢也会给大家展示几个例子，想要找到一些比较好的第二大脑数字花园的范例。最好的一个起点就是我们这个 obsidian 官方的同步平台，叫 obsidian think 的官网页面。因为官网页面下面会给大家提供一系列的范例，每一个范例都是一个博主花了很多时间精心打造的数字花园。大家有兴趣的话可以点进里面的内容来看一看。我在这里给大家挑了几个比较具体的例子啊。第一个例子就是我们这个网站上有一个针对 PARA 的展示。这个 PARA 的展示里会有针对 projects、areas、resources 和 archive 四个类别里各自的一些例子。比如说在 <span style="background:#fff88f">projects</span> 的类别里，<span style="background:#fff88f">personal（个人的）</span> projects 可能是一个计划去荷兰的旅行，然后 <span style="background:#fff88f">professional（职业的）</span> 可能是成为一个社交媒体的写手。然后 <span style="background:#fff88f">areas</span> 的话会比 projects <span style="background:#fff88f">更加长期，也更加宽泛一点</span>。所以 professional 可能就是视频制作或者是财务管理，然后公司的一些系统的管理等等。<span style="background:#fff88f">Resources</span> 的话可能是一些<span style="background:#fff88f">自己收集的资源</span>。然后最后一个就是 <span style="background:#fff88f">archives</span> 已经<span style="background:#fff88f">归档的内容</span>，这可能都是一些<span style="background:#fff88f">比较老的事情</span>了。比如说你之前计划了一个家庭旅行，所以这些信息你可以保存着，以后有需求的时候可以用。但是在大部分情况下，它就是一个被归档了的状态。这个呢就是 PARA 一个比较明显的例子。

然后我们除了 PARA，之前也介绍过 MOC，所以我们可以快速看一下 MOC 的例子。MOC 这里我用的是 LT Kit，LT 的全称是 Linking Your Thinking，是 YouTube 上一个比较有名的笔记博主，然后他在这里也分享了他的一整套的思维模式。我们这里最值得关注的是这个 LT 整体的一个思路。在点了几个键之后，你就可以立刻看到我们会进入一个 LT Kit MOC，每一个 MOC 里的内容都是一系列的双向链接。就比如说他在这里建立了一个 overview of the LT framework 框架的一个整体的介绍。在这个介绍里每一个模块都是更多的链接，能够让你详细的去探索里面的内容。这样就很方便用户能够在自己的这个网络里头去探索各种新鲜的事物。与此同时呢，你可以看到它是一个渐进式的情况。每一次你在探索新的内容的时候，都能够看到是什么样的内容把你引导到了现在这个页面里。

另外一个例子就是 Joshua 的笔记，然后我们可以看到他在这里也是给自己留了非常非常多的 MOC，比如说他有一个专门的读书 MOC，然后有一个兴趣 MOC。在这个兴趣 MOC 里头你还可以顺藤摸瓜地探索。比如说他这里有一个 self leadership 的反向链接，然后我们在 self leadership 底下又可以看到更多的 MOC，比如说有一个习惯 MOC、一个注意力 MOC 和一个记忆力 MOC。所以真的是有非常非常多的内容能够让你去探索，也同时保持了所有的内容之间都有强大的连接和整体感。在他的这个笔记里，你可以通过这个 graph 看到他这些笔记的状态。我们可以看到还是有非常非常多复杂的内容，形成了一个很酷炫的知识网络。

然后最后一个值得介绍一下的就是在国内博主这一块有一个 old winter 的数字花园。这个整体的建立思路和我们前面展示的几个范例都是差不多的。然后 old winter 它之前的内容对于我早期建立自己的知识库也是起到了一个非常大的启蒙作用。里面当然也有很多关于如何保持健康，如何进行个人知识管理，用好笔记工具，各种各样的双向链接方便大家去探索，所以也希望大家把这些作为参考。但是你自己最终建立起来的这么一个电子笔记的模板，会和这所有的范例都不一样。我觉得这也是一个非常有趣的部分。
