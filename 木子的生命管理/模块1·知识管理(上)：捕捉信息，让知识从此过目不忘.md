
各位暴风中的旅行者你们好，我是木子。欢迎来到生命管理课程的第二期。在上一期我们探讨了生命管理的四个模块，其中的一个就是知识管理。知识管理指的是把繁杂的外部信息转化成属于你自己的认知体系。在这一期里我们就来讨论一下内化知识的四部曲 ——Code 流程。所谓的 Code 流程来自于一个名叫 Tiago Forte 的博主，我们这两期的视频内容会参考他的书籍《打造第二大脑》，同时融入一些来自其他领域的经验，比如《认知觉醒》中的观点、卡片笔记写作法等等。

<span style="background:#fff88f">Code 代表 Capture、Organize、Distill 和 Express，也就是捕捉、整理、提取和输出。</span>我们先快速介绍一下每个阶段的意义。捕捉指的是在你每天看到、听到、想到各种信息的时候，适当去收集它们。比如收藏公众号的文字，记下转瞬即逝的灵感，这些都是捕捉的例子。而整理指的是把这些收集好的信息进行分类和归纳，像是给大脑准备一个书架，把散落在地上的书籍有序放进去。在这个过程中，大脑会给零散的信息开始建立结构和体系。第三步的提取指的是从整理好的信息里去掉冗余的部分，提炼要点。通过这种加工，原本属于别人的信息会脱胎换骨，逐渐变成你自己的东西。而最后的输出指的是将整理提取后的信息表达出来。比如你现在看到的这个教程就是我的一种输出。一个完整的 Code 流程，也被称为 Knowledge Loop（知识循环）。之所以被称之为循环，是因为输出后所获得的反馈会鼓励你去做更多的输入，带来一个无限进步的飞轮。比如说我平常看到有趣的视频和文章都会记到笔记里，然后把笔记里的内容做成视频之后，从观众那边收获的正反馈会赋予我更大的动力去读更多的书。

有很多人无法坚持记录知识，正是因为他们的 Code 四步没有凑齐。如果你光阅读不创作，那么你记录的东西就会堵在心里。这也是为什么我很鼓励大家多试试写作、做自媒体这类副业。Code 的每个阶段都有一些小技巧，我把它们总结成了关键词，了解这些关键词会让你更容易坚持知识管理。首先，Capture（捕捉）阶段的关键词是 “触动”，只有让你感到共鸣的事情才值得被捕捉。在《学会提问》这本书里，作者尼尔・布朗提到了人们在接受信息时存在的两种思维：全盘吸收的海绵式思维和主动筛选的淘金式思维。如果你像一块海绵一样对信息来者不拒，那么你的大脑马上就会饱和，被信息的洪流所吞没。毕竟你想一想，一个人每天能接触到的信息源，包括书籍、电影、电视剧、和他人的谈话、网上的课程、文章、视频、播客、博客等等。你自己的思维也是一个难以管理的信息源。美国国家科学基金会的数据显示，普通人的脑海里每天会闪过 1.2 万至 6 万个想法，这里面的每一种信息都在争夺你的注意力，但真正值得留下的只有凤毛麟角。所以你一定要学会挑剔、挑剔再挑剔，主动甄别哪些内容引发了你的强烈共鸣。举个例子，我们在读书的时候，往往会觉得：我打开了一本书，就应该努力把它读完。但美国经济学家泰勒・考文给了我们一个更好的选择。据说他读的十本书中，只有一本是从头到尾读完的，考文只记住他觉得有触动的部分，剩下的统统忽略。就像《认知觉醒》这本书里说的一样，面对生活中信息的滚滚洪流，“触动” 是最好的筛选器，它能让我们免受洪流的冲击，从而体面地行走在人间。

聊完 Capture，我们再聊聊 Organize（组织）。说到组织信息，你可能脑子里第一时间想到的是电脑里的文件夹，或者是一本书的目录。但你可以再回想一下，你现在脑子里的各种想法是按照一个个文件夹放好的吗？其实真正重要的并不是把信息归纳成列表，而是给知识之间创建关联。“关联” 就是我们 Organize 阶段的关键词。想要理解关联的重要性，我们就必须看看大脑背后的工作原理。每个人的大脑都有将近 860 亿个神经元，一个神经元会和若干个其他的神经元连接，通过中间的轴突传递电信号。如果你不断重复激活它们，一起被激活的神经元就会慢慢连接在一起，形成一条坚固的链路。换句话说，大脑储存和处理信息的过程，正是在这些神经元之间建立关联。作家周岭在《认知觉醒》这本书里是这么比喻的：孤立的知识就像沙砾，只有关联才能将其聚沙成塔，形成稳固的知识晶体，最终构建自己的认知体系。所以你每次在学一个新知识的时候，都可以想想：我在生活中有哪些和这个知识相关的体验呢？这些知识点可以应用在哪些领域呢？在其他的学科里有没有和这个知识点类似但名字不一样的概念呢？在你问这些问题的时候，你就正在把孤立的概念和生活经验创建关联，和你的其他知识创建关联。关联建立得越多，这个信息在你的脑子里也就记得越牢。

另外，现阶段无法建立关联的知识点，我们也要学会适当舍弃，哪怕你把它们硬塞到脑子里。因为没有关联，过不了多久，你也会忘掉。你的认知决定了你现在能够学习的极限。就比如说，你没办法让一个运动员瞬间学会理解金融市场的工作逻辑，但你也不用太担心，随着时间的推移，我们大部分人的知识网络都会变得越来越大，能够建立的关联也会越来越多。比如我们刚刚提到的这个运动员可能特别了解运动器械，于是退休后开了个器械公司，然后通过公司了解到记账的重要性，再通过记账逐渐接触到金融。通过一个个关联，原本的运动员也可以接触到一些从来没有设想过的知识。

接下来让我们探讨一些组织信息（也就是 Organize 信息）的具体方法。假设你最近在学拉丁舞，收集了不少相关的笔记，比如拉丁舞的步伐介绍、拉丁舞的历史概览、拉丁舞的比赛信息。那么接下来你该怎么整理这些信息呢？在这里给大家分享两个比较常见的组织信息的方式，一个叫 PARA，一个叫 MOC。PARA 的提出者还是我们刚才提到的 Tiago Forte。所谓的 PARA，它代表了 <span style="background:#fff88f">Projects（项目）</span>、<span style="background:#fff88f">Areas（领域）</span>、<span style="background:#fff88f">Resources（资源）</span>和 <span style="background:#fff88f">Archives（归档）</span>四个部分。Tiago 说，我们不应该按笔记的内容去分类，而是应该按笔记的功能去分类。第一个 Projects（项目）指的是近期具有特定目标和截止日期的事情，比如你要买个新电脑、要装修房间、要上网课等等。Areas（领域）指的是长期没有截止日期、需要持续维护的大课题，比如你的健康、你的理财、你要学习的市场营销、平面设计等等。Resources（资源）指的是你收集到的各种信息和知识，网上的各种文章、网课都是这样的例子。最后就是 Archives（归档），指的是当前用不上，但未来可能会有用的信息。PARA 框架能够描述一个人的思维从短期到长期的流动：短期的项目在完成之后，会根据性质被移动到领域和资源里；如果这个事情在中途废弃了，或者变得不再重要了，那么我们就可以放到归档中。拿我们的拉丁舞举个例子：你下个月要参加拉丁舞比赛，那可能是一个项目；而舞蹈可以算是一个长期持续的课题；你平常收集的很多关于拉丁舞的视频讲解和技术分析，可以看作是资源；拉丁舞比赛结束后，里面的报名信息之类的都已经不重要了，那么你就可以把它放到归档里。这是一个严谨的、框架化的、以行动为导向的整理体系。

那么除了 PARA 之外，我们还有一种更灵活的组织方式叫 MOC。MOC 的全称是 <span style="background:#fff88f">Map of Content（内容地图）</span>。一个 MOC 笔记就像是<span style="background:#fff88f">一个巨大的广场</span>，标注了<span style="background:#fff88f">各种指向其他笔记的通道</span>，这种通道也被称为<span style="background:#fff88f">双向链接（简称双链）</span>。简单来说，双链就是通过软件给两个笔记之间进行配对，这种互联互通的笔记体系有助于你发现知识之间的隐藏关联，形成一个更丰富的知识网络。我们在很多悬疑剧里会看到主角把一堆嫌疑人的线索用红线串起来，这其实就是在碎片化的信息之间建立关联。而 MOC 笔记就是一大堆双链的中心，MOC 的结构都是不固定的：一个心理学的 MOC 可能会分成理论研究、生活影响，在每个小标题底下各有一大堆其他笔记的双链；但一个拉丁舞的 MOC 就会截然不同，可能会分成步伐、音乐、练习计划等等。随着你对一个学科的了解不断深入，MOC 的形态也可以持续迭代，比如今天换掉一个小标题，明天改一改整体的结构，这些都是可以的。我个人比较喜欢灵活一点的记录模式，所以我的笔记体系主要基于 MOC；但倾向于系统记录的小伙伴也可以考虑一下 PARA。哦对了，还有一些人会喜欢用概念地图（Mind Mapping），通过更加视觉化的白板工具，更清晰地展示概念之间的关系。

我们最后要提一下，PARA、MOC、概念地图这些方法之间没有所谓的优劣，关键是要找到那种让你茅塞顿开，原本混乱的信息都梳理好、一切都通顺的感觉。在这节课里，我们探讨了 Capture 和 Organize 中的一些重点概念，以及 PARA、MOC 各自对应的关键词，还有 PARA、MOC、Mind Mapping 这三种负责 Organize 信息的方式。在后续的教程里，我们会介绍 Code 系统的后两个部分：Distill（提炼）和 Express（表达）。好的，各位，我们下节课见。
